

# Fix Interieur Prompt: AutoCity verwijderen + Camera schermen beschermen

## Probleem

De huidige `buildInteriorPrompt()` heeft twee concrete fouten:
1. **"AutoCity" staat letterlijk in de prompt** (regels 346 en 359) — het model leest dit en tekent het op muren/vloeren
2. **Camera schermen niet expliciet beschermd** — achteruitrijcamera, 360-camera displays worden niet herkend als "behouden" content, waardoor AI ze kan overschrijven

## Wijzigingen in `supabase/functions/showroom-photo-studio/index.ts`

### 1. `buildInteriorPrompt()` — Prompt herschrijven

**Regel 346** — verwijder "AutoCity":
```
"The AutoCity photo booth environment" → "The photo booth environment"
```

**Regel 359** — verwijder "AUTOCITY":
```
"AUTOCITY PHOTO BOOTH (10m x 8m x 4m hoog):" → "PHOTO BOOTH OMGEVING (10m x 8m x 4m hoog):"
```

**Nieuw blok toevoegen na HARDE EIS #3** — Camera schermen beschermen:
```
HARDE EIS #4 — CAMERA SCHERMEN ONGEWIJZIGD:
If the infotainment screen shows a BACKUP CAMERA view, 360-DEGREE CAMERA view, 
parking sensors visualization, or ANY camera feed: preserve it EXACTLY as-is.
Do NOT replace camera feeds with a black/dark screen.
Do NOT alter the content shown on ANY active display.
If a screen shows navigation, media, or settings UI: preserve ALL text, icons, and layout exactly.
```

**Versterk de retouche-sectie** — maak expliciet dat het gaat om opschonen, niet opnieuw genereren:
```
RETOUCHE — OPSCHONEN, NIET OPNIEUW GENEREREN:
You are a RETOUCHER, not a CREATOR. Every pixel of the vehicle interior must come from the input photo.
- Remove dust, dirt, fingerprints from all surfaces
- Clean leather/fabric: remove stains, refresh texture, sharpen stitching
- Dashboard/trim: remove fingerprints, condition to matte finish  
- Replace harsh outdoor lighting with soft warm 3000K studio lighting
- Result: 0 km showroom quality, but the SAME vehicle, not a similar one
```

### 2. Response parsing robuuster maken (regels 309-317)

De huidige code filtert alleen op `type === "image_generation_call"`. Toevoegen van debug logging en flexibele extractie:

```typescript
const result = await response.json();
console.log("Response output types:", JSON.stringify(result.output?.map((o: any) => o.type)));

const imageData = result.output
  ?.filter((o: any) => o.type === "image_generation_call")
  ?.map((o: any) => o.result);

if (!imageData?.length) {
  // Fallback: probeer andere output types
  const altImage = result.output?.find((o: any) => o.type === "image");
  if (altImage?.data) return altImage.data;
  console.error("Full response output:", JSON.stringify(result.output?.map((o: any) => ({ type: o.type, keys: Object.keys(o) }))));
  throw new Error("No image generated by Responses API");
}
```

### 3. Prompt logging toevoegen (tijdelijk, voor debugging)

Na regel 497 (`const prompt = buildInteriorPrompt()`):
```typescript
console.log("INTERIOR PROMPT (first 600 chars):", prompt.substring(0, 600));
```

Dit helpt om te verifiëren dat de juiste prompt daadwerkelijk wordt verstuurd.
